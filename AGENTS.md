# AGENTS.md

このリポジトリにおける AI Agent / 自動化ツールの振る舞いルールを定義する。

## 🎯 Goal

- 最小コストで検証環境を構築する
- 人間の意思決定に必要な情報を整理する
- 再現性のある検証を行う

## 📁 ディレクトリルール

### experiments/

- 単一ライブラリの検証を行う
- 不要な依存は追加しない
- 最小構成で開始する

### integrations/

- 複数ライブラリの組み合わせを検証する
- 実運用を想定した構成にする
- 適切なディレクトリ構造を持たせる

## 🛠️ プロジェクト作成ルール

### 必須

- 各プロジェクトに `NOTE.md` を作成する
- package manager はプロジェクトごとに自由

### 初期セットアップ

- 可能な限り公式の推奨方法を使う
- 不要な boilerplate は削除する
- 起動確認まで行う

## 📝 NOTE.md 更新ルール

Agent は以下を必ず記録すること：

### 1. セットアップ手順

- 実行コマンド
- 必要な環境
- 初期化方法

### 2. 実験内容

- 何を試したか
- どの設定を変更したか

### 3. 結果

- 成功 / 失敗
- 挙動の違い

### 4. 学び

- 良い点
- 悪い点
- 注意点

## ⚙️ コーディング方針

- シンプルを最優先
- 検証に不要な abstraction は作らない
- コメントは最小限（NOTE.md に書く）

## 🔄 更新方針

- 既存プロジェクトを壊さない
- 変更は最小単位で行う
- 比較可能性を維持する

## 🚫 禁止事項

- グローバル設定の変更
- 無関係な依存の追加
- NOTE.md を更新しない変更

## 🤝 人間との役割分担

### Agent

- セットアップ
- 実験
- 記録

### Human

- 評価
- 意思決定
- 採用判断

## 🧭 判断基準

迷った場合は以下を優先：

1. 再現性
2. シンプルさ
3. 比較しやすさ

## 📌 補足

このリポジトリは「検証」が目的であり、「完成度」は優先しない。
